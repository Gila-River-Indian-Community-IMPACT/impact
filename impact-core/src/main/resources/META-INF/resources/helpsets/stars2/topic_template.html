Overview (This is a level 2 Heading)

(In a "real" topic page, this section provides an overview of the purpose & scope of the page from which this help file was accessed.)

The overview of this file provides by example most, if not all, of the directives the system needs in order to produce the help topic page(s) you will write. It uses cascading stylesheets to implement standard "Browser Look-and-Feel" (or blaf) formatting, overriding it when appropriate with styles that mimic and/or blend with the styles that Stars2 uses. The file demonstrates Bookmarks, Font Styles, Prose Text, Numbered and Bulleted Lists, Graphics, and Tables.

You can open this file using the KompoZer tool, and edit the view that KompoZer provides. Doing so modifies the HTML information that the browser uses to render the page. KompoZer gives you controls similar to MS Word with which to add graphics and links to your page as well as to provide information in numbered and unnumbered lists, and in other formats. You can also 'preview' your page within KompoZer; although it may not look exactly like you expect. To see what the page will really look like in any particular browser, be sure that the appropriate stylesheets (blafdoc.css and stars2.css) are in your development directory and open the file with the browser you want to preview.  Most of the "instructions" presented in this template file are descriptions of how to accomplish things within the KompoZer tool.  You may also use other tools or even edit or "tweak" the raw html source code that is produced by KompoZer (or another tool), if you prefer.

For testing purposes, you will need to have your image files in a directory called "images" under the directory where you are storing your HTML (topic page) files. If you're using KompoZer, this will most conveniently be in the Program Files\KompoZer directory. If the images you are using are shared among multiple pages, then you should copy the image files you need from their "official" home into your development directory; if you are using images unique to your page, then be sure to provide your image file when you incorporate your page into the 'help' infrastructure so that it will be available from the "official" page when that page is accessed and also so that it can be made available to others for use in their pages as well. We will copy the template file (topic_template.html), the stylesheets, shared image files and
the J:\SHARED\UNICON\Help Source Files directory with the stylesheets and shared image files you will need.

Note that the spacing around headings (regardless of level) and between sections are hard-coded --- it is NOT defined as part of the style. It is, however, included in this template file, so careful text replacement should preserve this spacing.

Graphics& Images (Another level 2 Heading)

Images may be easily inserted in your page using the 'Insert' menu, and/or by selecting the 'Image' icon on the toolbar.  From either of these points, you specify the filename of the image file.  You are required to provide 'Alternate text', which is the text that appears in the designated spot if your browser doesn't support such images .  You may also provide 'Tooltip' text.  This is optional.  Any text you provide in this field appears if the user places their cursor over the image and holds it there for a brief period of time.  

It's preferable that you use .gif files for any graphics you which to include on your page, as they require considerably less storage space than other graphics formats.  Create an "images" folder in the folder in which your topic page files (the .html files) are stored.  Then when you insert a link into your page, the link reference will be "images/<image_file_name>".

Glossary Links

You can establish glossary-links to words in your page, but it must be done manually (i.e., KompoZer doesn't currently support it). To do this, insert the following tag around the word you want to link to its glossary entry, here called your TARGET_TERM:

<a class="glossary-link" href="glossary.html/#TARGET_TERM" target="_blank">TARGET_TERM</a>

And, add an entry for the term in the glossary.html file.

<dt><a name="TARGET_TERM">Target Term</a></dt>
<dd>Specify the definition text for target_term here</dd>

(The </dd> tag at the end of the 'dictionary definition' text is optional.)

System Glossary

Introduction

This glossary presents definitions and explanations of terms that you may encounter in the system.  It covers acronyms and abbreviations as well as full terms for which such information adds to the usability of the system.

At AQD's discretion, this glossary may be segregated by 'first letter' or by overarching topic or all the terms may be combined in whatever order they are specified.


Glossary Terms

Banner
The top section of every screen in the IMPACT system. This section always contains the Ohio EPA logo, a brief welcome message identifying the person who is logged in to that screen, and three common hyperlinks that are available on every screen to make it easy for you to get to locations within the website that you may commonly wish to visit.
AQD
Division of Air Pollution Control.
The division of the Ohio Environmental Protection Agency that manages applications for, produces and issues permits with respect to emissions and also monitors compliance on the basis of reports submitted by RCUs on behalf of their facilities.
Datagrid
The name used in this system for a table of data, often the result of a search you execute from a search window. These tables have special cabailities, such as paging, sorting, and also provide access to operations that create a printable view of the data or export the data to Excel.
DO/LAA
District Office/Local Area Agency.
The offices from which personnel provide air services to RCUs. These offices may be organizationally housed within the Ohio EPA (DOs) or may be independent of the agency (LAAs). Each DO/LAA is responsible for RCUs located in their specified region; in total, the DO/LAAs "cover" the entire state.
Facility ID
First-level menu
This term refers to the set of tabs that appear across the top of each content or operational screen in the system. They give you access to the various sub-system functionality that the system provides. (These are not the same tabs that appear on the Help topic pages.)
Hyperlinks
Hyperlinks are items on a screen that, when clicked, take you to another, presumably related, screen. Hyperlinks normally appear as underlined words or sets of words, where the words give an indication of the target or destination to which you are taken when you select them. The color and other behavior of hyperlinks depends upon the browser you are using to access the web pages.
Menu
A well-defined set of values for fields or operations among which a user may choose. It can be used to refer to the tabs at the top of each screen, also referred to as a first-level menu, the options offered just below the tab on each system screen, referred to as a second-level menu, the operations, if any, listed along the left side of each screen, a third-level menu, or the choices available from which to select a value for a field on an input form, also referred to as a pick-list.
Second-level menu
This term refers to the set of operation choices that appear on a line just below the tabs, or first-level menu items on a screen. These items differ depending upon which tab is selected. They give you access to the functionality that the system provides under each tab.
Paging
This term describes the capability that the system provides to control which rows in a datagrid you can see on the screen at any point in time. Initially, a table shows the first set of rows. If the table doesn't have more rows, then you will not see the paging controls. However when a table of data is much longer than you can see on a single screen, the system provides controls you can use to move among the 'pages' of data in the table. The number of rows that a datagrid shows on a single screen is also the increment by which it pages.
RCU
Regulated Community User.
The owners, operators and/or other delegated users of the IMPACT System who are associated with facilities that have a relationship with AQD with respect to permits for air emissions and the enforcement thereof. These users access the system via the Ohio EPA e-Business Center 
Sorting
This term describes the capability that the system provides to sort the rows in a datagrid in either ascending or descending order by any column in the data. Initially, a table appears with a default sort order --- most often ascending by the numerical identifier for the objects represented by the data. However, you can click on any column header to sort by the data values in that column. The system indicates the direction of the sort with an up (for ascending) or down (for descending) arrow in the column header.
Stars2
This system, which supports the Ohio EPA Division of Air Pollution Control and its Regulated Community Users in managing Air Pollution Control permits and emissions reporting. The system provides a web interface for AQD personnel as well as for RCUs (via the Ohio EPA e-Business Center) to input information and manage the permit generation and tracking processes. Note that the name of the system that is used internally within AQD (Stars2) will never appear to the outside user; they will know the system as "e-Business Gateway - Air Services".

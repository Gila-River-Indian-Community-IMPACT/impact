Reassign a Workflow

Overview 

The third-level menu Reassign of the Workflow Diagram second-level menu contains a datagrid with all tasks associated with a workflow.  The purpose of the Reassign Tasks page is to allow those with rights to assign a different user to a task(s).  For more information on rights and user roles, see the Help page entitled “Facility-Based User Roles.” 

You will see a Task List datagrid of all tasks with the following columns of data: Process ID,  Task Name, Task Status and Staff.  If you have rights to assign user’s tasks, the Staff column will contain a pick list for each task.  The default name is the current user assigned the task.  You can select another name from the pick list and click Apply to reassign the task to the selected user.

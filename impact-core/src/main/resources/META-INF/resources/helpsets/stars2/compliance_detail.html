Compliance Report Detail

Overview

This topic covers the second-level menu, Compliance Report Detail, under the Compliance Reports tab. Compliance reports are either completed by the owner or operator of a facility manually (hard copy) or electronically. Hard copy submittals are entered electronically by DO/LAA staff. The same compliance report forms are completed regardless of whether they are entered by DO/LAA staff from hard copy reports or submitted electronically through the gateway by a regulated community user. One exception is that the TV Annual Compliance Certification is required to be submitted electronically by the external user.

Depending on whether you are viewing a report that has been submitted or you are viewing a report that has yet to be submitted “electronically” (i.e., a report in the process of being electronically entered by DO/LAA staff), the report may be editable through the Compliance Report Detail page. The only reports that an internal user can view are those that are submitted from an external user, or those that were submitted in hard copy form and are entered electronically by DO/LAA staff.Once the entire report has been validated and submitted, only portions of the report will be editable as part of the review process performed by DO/LAA staff.

This Help page will focus on the fields you view on the Compliance Report Detail page, fields that are editable IF you are entering a hard copy report, or fields you will edit during the review process. Certain steps may need to be completed in other sections of the system in order to access a report through the Compliance Report Detail page for continued editing. These steps are discussed briefly under the heading “How Are Compliance Reports Electronically Entered by the DO/LAA?” below. The end of this Help page provides links to several “Related Help Topics” that will allow direct access to additional Help you may need during this process.

How Are Compliance Reports Electronically Entered by the DO/LAA?

Recall, DO/LAA staff will enter hard copy reports submitted by the owner or operator of a facility. In order to begin report entry the following steps are necessary:

  • Facility Profile: confirm existence or create

    Under the Facilities tab perform a Facility Search using the Facility ID. If you are unable to locate the Facility ID in the system, it is likely the facility has not yet been created within the system and a report cannot be entered until this is done. You must create the facility using the second-level menu Create Facility of the Facilities tab. This is the first step of creating a Facility Detail. Please refer to the Help page for Create Facility for further important information concerning creating a facility.

  • Facility Profile  Compliance Report(s): begin report entry

    Once you have created, if necessary, and accessed the Facility Detail page of the requesting facility, select the third-level menu Compliance Report(s) link. The initiation of report entry occurs in this section of the system. The instant you generate a report through this link the system generates a Compliance Report Detail page. For Help on beginning the report entry, please refer to the Compliance Report(s) Help page for more information.

Locating and Viewing an Electronic Compliance Report

You can locate a report and access a Compliance Report Detail page throughout many of the system pages, most usually through the Compliance Report Search or a Facility Detail pages.

  • Facility Profile  Compliance Report(s) third-level menu will display a datagrid of all reports for the specific facility.

  • Compliance Report Search allows you to search for available reports using many different search criteria. This feature is available on the opening page for the first-level menu called Compliance Reports. For Help on this feature, refer to the Compliance Report Search Help page. Reports that fit the search criteria you have set will show up in the Search Results datagrid after you have performed a Compliance Report Search.

Using either method, the datagrid will contain a column with the Report ID which is hyperlinked to the specific Compliance Report Detail page. Click on the Report ID.

Validation and Submitting –  and 

Recall, only Permit Evaluation Reports and Other Compliance Reports may be entered electronically, and submitted, by DO/LAA staff when a hard copy form is received from the facility. When you are ready to submit the completed report, it will need to pass a validation process to ensure the minimum information is included. These validation checks apply equally to internal and external submittals. Therefore, if you are entering a hard copy report it is important that all information contained in the report is entered before submission. To Validate click  at the bottom of the page. A pop-up box will appear displaying any issues associated with the validation. The issue(s) must be resolved in order for report submittal to occur.

Issues with validation are categorized as either ERRORs, WARNINGs or INFOs. ERRORs must be resolved before the document can be validated. If you only experience WARNINGs or INFOs, the validation process will be successful without the need to click validate again; however, WARNINGs and INFOs may be viewed and corrected/changed before submission. If changes are made after validation, it will be necessary to validate again.

To navigate to any ERRORs, WARNINGs or INFOs, click on the brown underlined hyperlink associated with the issue. This will take you to the page where it was detected. The page should open in the edit mode. If it is an error, you will need to correct the error and navigate back to the Application Detail page to validate again or open the validation pop-up using the toolbar at the bottom of your computer screen and move on to the next error message hyperlink until all errors have been corrected. WARNINGs and INFOs may be viewed by following the above procedure. When viewing, you can determine if the information requires a change or if you want to keep the information as originally determined. WARNINGs typically may be more significant issues than ERRORs.

Once the document passes validation, you will see the Information- validation successful message at the top of the screen. Once all issues are resolved, click  to complete the process.

TV Annual Compliance Certification

This section explains the fields you will see in a TV Annual Compliance Certification and fields that are completed as part of the process of reviewing a report. Please note, Title V facilities will be required to submit their TV Annual Compliance Certification electronically in the system; therefore, you will not be able to create a TV Annual Compliance Certification. Title V facilities have two options available for fulfilling the electronic submittal requirement. They may choose to complete the Identification of Intermittent Compliance (IC) (including any necessary attachments) and Any Material Information Not Established Through the Applicable Permit Terms and Conditions That May Indicate Non-Compliance sections or they may choose to use their own form that fulfills the requirements for a TV Annual Compliance Certification and attach the document in the Attachments section of the system’s form.

  • AQD Staff Section

    • Received On: This field identifies the date the application was electronically submitted by an external user.

    • Reviewed On: Enter the date you performed a review of the contents of the report.
    • Reviewed By: This field will default to the name of the person performing the review when the Reviewed On date is entered. It is not editable; therefore, if you are not reviewing a report you should not enter a Reviewed On date.

    • Report Accepted: Upon submittal this field will default to Not Yet Reviewed. When reviewing, it must be changed to Yes, No, or No-Revision Requested. Keep in mind, selecting an acceptance type does not generate a workflow or directly notify a facility. For example, if No-Revision Requested is selected, DO/LAA staff will need to take appropriate action outside of the system to request the revision.

    • Deviations Reported: Upon submittal this field will be blank. When reviewing, it must be changed to Yes or No.

    • Comments: This editable field accepts free text entry for any important information the person reviewing the report may need to convey.
  • Identification of Intermittent Compliance(IC) Section

    This section contains a datagrid of information regarding any deviations or excursions being reported if the owner or operator of the facility is choosing this reporting option over the attachment option.

    • IC ID: Unique ID generated by the system for the associated intermittent compliance row.

    • EU ID: The emissions unit for which the deviation or excursion is being reported.

    • Emission Limitation/Control Measure or Permit Term No. and Compliance Method : Additional information about the contents of this fields can be found in the “Title V Compliance Certification Form and Instructions” found in the External References section of the system.

    • Excursions/Deviations (one of the following must be provided): For each excursion or deviation, either the column marked Report Date of Those Documented Within Excursion/Deviation Reports Submitted to DO/LAA OR the column marked Explain the Date, Nature, Duration, and Probable Cause of the Excursion/Deviation, as well as any Corrective Action Taken will have been completed, but not both. If the facility chose to complete the Report Date of Those Documented Within Excursion/Deviation Reports Submitted to DO/LAA, they may also include the report as an attachment in the Attachment section (discussed below). Additional information about the contents of this fields can be found in the “Title V Compliance Certification Form and Instructions” found in the External References section of the system.
  • Any Material Information Not Established Through the Applicable Permit Terms and Conditions That May Indicate Non-Compliance Section

    The owner or operator of the facility will have reported any necessary information per the instructions in this section. Additional information about the contents of this fields can be found in the “Title V Compliance Certification Form and Instructions” found in the External References section of the system.

  • Attachments Section

    The Attachments section is used to incorporate any Attachments submitted as part of the report, including if the owner or operator of the facility is choosing to submit their own form as an attachment rather than completing the other fields of this report. The Attachment Type field is a hyperlink that allows you to access and view the attachment.

Permit Evaluation Report

This section explains the fields you will see in a Permit Evaluation Report, fields to complete when you are entering a hard copy report received, and fields that are completed as part of the process of reviewing a report.

  • AQD Staff Section
    • Received On: This field identifies the date the report was electronically submitted by an external user, or if entry is occurring by DO/LAA staff, enter the received by Ohio EPA date (e.g., post mark date).

    • Reviewed On (only viewable after submission): Enter the date you performed a review of the contents of the report.
    • Reviewed By (only viewable after submission): This field will default to the name of the person performing the review when the Reviewed On date is entered. It is not editable; therefore, if you are not reviewing a report you should not enter a Reviewed On date.

    • Report Accepted: This field will default to Not Yet Reviewed. When reviewing it must be changed to Yes, No, or No-Revision Requested. If entry is occurring by DO/LAA staff you will only be able to edit this field after electronic submission occurs. Keep in mind, selecting an acceptance type does not generate a workflow or directly notify a facility. For example, if No-Revision Requested is selected, DO/LAA staff will need to take appropriate action outside of the system to request the revision.

    • Deviations Reported: Upon submittal this field will be blank. When reviewing, it must be changed to Yes or No.

    • Comments: This editable field accepts free text entry for any important information the person reviewing the report may need to convey.
  • Additional Information & Corrections Section

    This section contains additional information the reporter may need to communicate. If you are entering a hard copy report it will correspond to the same section in the hard copy form. Additional information about the contents of this field can be found in the “PER FAQs” document found in the External References section of the system.

  • Detailed EU Information Section

    This section contains a datagrid of information regarding whether or not deviations or excursions occurred for each emissions unit issued a PTIO. If you are entering a hard copy report it will correspond to the same section in the hard copy form.

    • AQD EU ID: The emissions unit identification for which a PER is required. This is pre-populated based on system’s identification of emissions units at the facility that have been issued a PTIO.

    • AQD EU Description: The emissions unit description for which a PER is required. This is pre-populated based on system’s identification of emissions units at the facility that have been issued a PTIO.

    • Permit Number: The PTIO permit number containing the emissions unit’s requirements. This is pre-populated based on system’s identification of emissions units at the facility that have been issued a PTIO.

    • Dates:

      The following dates are pre-populated based on information contained within the Facility Profile for each emissions unit. If the information is contained in the Facility Profile, it will be pre-populated in this section and also in the hard copy form provided to the owner or operator of the facility. If any information is missing but available at the time the report is to be submitted, it should be entered into the corresponding Emissions Unit Information page of the Facility Profile (see the Facility Tree for Help page for assistance on editing an Emissions Unit Information page). As indicated in the instructions on the screen, “Failure to update your Facility Detail may result in an incomplete report submittal.”
      • Completion of Initial Installation

      • Begin Installation/ Modification

      • Commence Operation after Installation or Latest Modification
    • Deviations or Exceedances From:
      • OR/EL: Will identify, or select from the pick list when entering a hard copy, Yes or No for any deviations or exceedances from operational restrictions(OR)/emission limitations(EL).

      • MRR: Will identify, or select from the pick list when entering a hard copy, Yes or No for any deviations or exceedances from monitoring, recordkeeping or reporting(MRR).

      If Yes is identified, either an attachment should be included in the Attachments section OR the Deviation, Exceedance, or Visible Emissions Detail datagrid must be completed identifying the additional information required for each deviation or exceedance. Additional information about the contents of this field can be found in the “PER FAQs” document found in the External References section of the system.

  • Notes: this section may be used to identify any information regarding the deviations or exceedance.
  • Deviation, Exceedance, or Visible Emissions Details Section

    Additional information is required for each deviation or exceedance that prompted a Yes answer in the Detailed EU Information datagrid and for any visible emissions (VE) incident that occurred during the reporting period. In lieu of completing this datagrid, information that fulfills the requirements can be included as an attachment in the Attachments section. Additional information about the contents of this field can be found in the “PER FAQs” document found in the External References section of the system.

    The datagrid contains the following fields, which when entering a hard copy form require entry of the fields identified in the pop-up below generated by clicking on .

    • Dev/Exc/VE ID: Unique ID generated by the system for the associated deviation/exceedance/visible emissions row.

    • AQD EU ID: The emissions unit identification for which Yes was indicated. When entering a hard copy, this is entered into the pop-up identified above.

    • Start Date: Identifies the Start Date of the visible emissions (VE) incident, exceedance or deviation. When entering a hard copy, this is entered into the pop-up identified above.

    • End Date: Identify the End Date of the visible emissions (VE) incident, exceedance or deviation. When entering a hard copy, this is entered into the pop-up identified above.

    • Description of Deviation or Exceedance AND Probable Cause and Description of Corrective Actions. If none, describe why not: When entering a hard copy, this is entered into the pop-up identified above. Additional information about the contents of this field can be found in the “PER FAQs” document found in the External References section of the system.

    When entering a hard copy report, click  to populate the information into the datagrid or click  to return to specific Compliance Report Detail page. Clicking  will cancel any report information you just entered electronically.

  • Attachments Section

    The Attachments section is used to incorporate any Attachments needed to complete the report. This includes the requirement to attach additional information for each deviation or exceedance that prompted a Yes answer in the Detailed EU Information datagrid and for any visible emissions (VE) incident that occurred during the reporting period when the Deviation, Exceedance, or Visible Emissions Details Section was not used to report such information.

    Click  which will generate the following pop-up window:

  • Attachment Type: Select either Other, Process Flow Diagram(s) or PTIO Permit Evaluation Report (PER) attachments from the pick list.

  • Description: Enter an appropriate Description of the attachment.

  • File to upload: Choose the file you would like to attach to the submittal from your computer.

    Click  to populate the information into the datagrid or click  to return to specific Compliance Report Detail page. Clicking  will cancel any report information you just entered electronically.

    The datagrid will be automatically populated with an Attachment ID (unique ID number automatically generated by the system and hyperlink for editing the Attachment Type or Description or deleting the attachment), Attachment Type (hyperlink to access attachment), Description, Modified By and Last Modified (date the attachment was last uploaded).

Common Search Paradigm

Overview

Navigation within most tabs in the IMPACT System begins at a Search screen. The screen allows you to access a set of objects based on criteria you specify in the Search Criteria region of the screen. Different search criteria apply to different searches according to the tab you are in, and therefore the objects you will be searching. The system provides operations buttons that allow you to execute your search when you are ready. An executed search returns a table of data known as a datagrid.

This help topic contains the following sections:

  • Search Criteria Basics
  • Using Wildcards
  • Search Button Operations
  • Search Results
  • Related Help Topics

Some of the information discussed here, in particular input fields, also apply elsewhere in the system. Therefore, they are covered in greater depth in other help topics within the Common User Interface Elements chapter of On-line Help.

Search Criteria Basics

Most often, you specify the search criteria that define the set of objects you are looking for in a Search Criteria window that the system presents when you select the tab for any searchable objects, such as Facilitites or Invoices (Figure 1).  Some search criteria fields have default values; that is, when the window first appears, some of the fields have values already in them.



Figure 1: Example Search Criteria Window

You can, of course, change any of the default values that you want. You may complete none, one, several or all input fields for a particular search. If you leave any criterion input field blank, the system assumes you want items that match any value for that criterion. Searches apply the criteria you specify cumulatively, so that if you specify, for example a County and an Operating Status for a Facility Search, the search will return only the Operating facilities within the specified county.

There are several types of input fields that may appear in a Search Criteria window. Each possible input field type is described in great detail on the Input Fields Help page. For searches, you will most frequently encounter the types listed below:

  • Text input fields - A text input field consists of a label, identifying the information that the system expects you to type in, and a field into which you can type. Certain fields such as Facility ID and Facility Name shown above support the use of wildcard characters within the input you type. (See Using Wildcards)
  • Fields offering pick lists - you select one value from the list list of the choices you have for values in that field. You access the list of choices by clicking at the end of the input field. 
  • Date fields - you can type in a date string in nearly any format or select the date you want from the calendar that pops up when you select the icon that the system offers at the end of the input field.

Using wildcards

The system supports the use of two wildcard characters, asterisk () and percent sign (%). You may use either of these characters at any location within a string in any text input field that supports them. Either wildcard character, at any point in a string you type, represents zero or more other characters, no matter what those characters are. All Facility ID and Facility Name fields in the system, as well as select other fields, support this functionality. For example, in a Facility Name field,

Uni*             matches 272 facilities, including  United Dairy Farmers, Union Tools Inc, University of Dayton Unicoy Alloys and Metal 

*Metal*        matches 150 facilities, including  Unicoy Alloys and Metal, National Metal Finishing, Metal Coatings Intl, Metaldyne

U*Metal%     matches 6 facilities, including Unicoy Alloys and Metal, Universal Metal Products, U S Metalcraft, USG Interiors - American Metals Corp.

Unico*          matches 2 facilities, 2 locations of Unico Alloys and Metal

In a Facility ID field,

*01*              matches 5919 facilities, including facilities with IDs of 0120001226, 0141000234, 02100090560 

01*               matches 1866 facilities, including facilities with IDs of 0120001226, 0120000222, 0125040590

012%6          matches 140 facilities, including facilities with IDs of 0120001226, 0121000126, 0121000226, 01250040026

012%00*26   matches 12 facilities, including facilities with IDs of 0121000126, 0121000226

Use of wildcards does not, of course, preclude specifying other criteria for your search. For example if you specify a Facility Name and/or Facility ID that contains wildcards, and you also select 'Franklin' for the County field, then your search will return all the facilities that match your wildcarded criteria that are in Franklin County. As you can see, wildcards give you considerable flexibility in the way you construct your search criteria, and thus control the set of records that your search returns.

Search Button Operations

Every search window includes two buttons, and .

After completing all the search criteria you choose to specify, click , which instructs the system to execute your search; it applies criteria you specify cumulatively. That is, the system only returns items that match all the values you specify.

You can modify your search criteria and re-execute your search, which you will often want to do to narrow the search to return fewer records and/or to conduct a search on a different set of criteria. You can, of course change just one or two criteria before you re-execute the search. (Click. after the criteria you want to change.) Somestimes, however, you may want to start your search all over again with a completely new set of criteria. Click to clear all the values you have entered and return all the search criteria input fields to the default criteria values.


Search Results

No matter what you are searching for (Facilities, Permits, Emissions Inventories, etc.), when you execute a search, the system returns a table, each row of which represents one of the objects. The table is also known as a datagrid. Figure 2 illustrates the datagrid the system returns when you search for the facilities in Franklin County, serviced by the Central District Office which have names beginning with 'A'.


Figure 2: Example Search Results Datagrid
Datagrids provide you with significant functionality. They can help you locate detailed data about any one object listed in the table, and more. The Datagrid Help page provides more information on the capabilities that datagrids give you.

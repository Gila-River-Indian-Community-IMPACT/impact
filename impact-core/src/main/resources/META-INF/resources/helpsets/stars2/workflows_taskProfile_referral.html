Refer a Task

Overview 

The third-level menu, Referral, of the Task Profile page allows you to identify an entity who you have asked for additional information from in order to continue processing the task.  You can refer only tasks that are 'In process'.   Referring a task does not change the “In Process” status of the task or the user to whom the task is assigned but the task will disappear from the assigned user’s ToDo List until the referral period ends.  There is no limit to how many times you can refer a task and a new referral may begin immediately after a referral has ended.  If you are concerned about forgetting about the task, you can generate a substitute task for yourself by using the Create ToDo Task on the second level menu of the Home tab.

Referring a Task

      The following selections are required in order to refer a task:  

 

  • Referral Type: select the entity whom you are referring to from the picklist.
  • Extend Process Due Date: provides the ability to lengthen the Task’s due date by selecting Yes from the pick list.  As each day passes, the Due Date will increases automatically.  This does not affect the ultimate due date for the workflow itself.  Unless a facility Referral Type is selected this box will automatically default to No.  Once a facility Referral Type is selected it will automatically default to Yes.  They system will automatically extend the workflow due date for facility referral’s equivalent to the number of days it is in referral.
  • Referral End Date:  identify an end date for the period of time necessary to obtain the additional information from the entity.  This is the date on which the referral will automatically end and the task will reappear on the user’s ToDo List.  When a specific date is not selected, a default value will automatically be created, as identified in the table below.

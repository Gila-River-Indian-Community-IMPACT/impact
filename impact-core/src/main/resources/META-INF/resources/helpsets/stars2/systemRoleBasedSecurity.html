System Role-Based Security

Overview

When you are provided access to the system as a user, you not only receive a login name and password, you are granted one or more specific User Roles . Each role has a defined set of rights, allowing those users to perform a specific set of functions. If you do not have rights to perform a specific function, you will either not be able to see the function or the function will be displayed but unavailable for you to select. For instance, Administrators can modify pick-lists throughout the system in the Admin tab which is not visible to other users.

There are user groups associated with general permitting duties, permit issuance, trade secret access and supervisory duties, among others. Should you need increased access to specific system functions, such as having the ability to view and print Trade Secret documents, contact a system Administrator.

Assigning User Roles

Each facility has 16 specific Facility Roles, which are viewable via the User Roles third-level menu in a Facility Profile. Staff have been assigned these roles based upon input from each office. The assignment of Facility Roles is editable by a supervisor or any other person with the appropriate System Role. For example, every facility is assigned a DO/LAA Permit Writer to perform the preliminary and technical review of their permits and reports; a DO/LAA Reviewer/Approver to sign off on those permits at the DO/LAA; and a CO Issuance Coordinator to issue permits. See the User Roles help page for information on this feature.

When a workflow is initiated (ex.permit application submitted), the system’s workflow engine builds the workflow and pre-assigns all possible tasks in that workflow to the specific user who has the associated role for the facility that is inherently generating the workflow (permit applicant).

Once the user roles are changed in the facility inventory, all subsequent workflow tasks will be assigned to the new staff person that has the role. Workflows that were already established prior to the reassignment will retain the previously assigned personnel. These previously assigned personnel can be changed by going to the specific workflow and reassigning either individual tasks or the entire workflow. Navigate to the workflow and click Help in the upper right hand corner of the screen for more details concerning reassignment of the workflow. For help on this feature, view the help page entitled Reassign of the Workflow Diagram.

Also available to a supervisor is the ability to make wholesale reassignments via the second-level menu of the Tools tab (Facility Operations > Facility Roles).

New Facility – Role assignments

When a new facility is created in the system, Facility Roles are established based on a definition table maintained by the System Administrator. The assignments are made on a county level. Contact a System Administrator to have the default assignments modified.

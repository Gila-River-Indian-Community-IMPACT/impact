Permit Detail

Overview

The Permit Detail page gives you several options for viewing details of a permit that has been issued or is in the process of being issued, and for those in process, the ability to edit data that will be relevant to issued documents. When Ohio EPA receives, or enters, an application electronically, a corresponding Permit Detail page is generated. Only those emissions units that are included in the application (not always all emissions units at the facility) will be identified in the corresponding Permit Detail page. The following types of permits are available for viewing and, when applicable, editing: PTI (for a facility subject to the Title V program), PTIO (for a facility subject to the non-Title V program), Registration (historical), State PTO (historical), and TV PTO. All Permit Detail pages contain the following sections: a Facility Tree, Permit Information (Registration Status for historical Registrations), Terms and Conditions, and Notes. The contents of the Permit Detail page will differ depending on the type of permit you are working with (e.g., Title V versus PTIO).

At the top of the Permit Detail page you will see and review basic facility information and information on the permit’s status. Information included in the heading is the Permit Type, associated Permit Number and issuance dates of the Draft, PPP, PP, and Final permit documents, where N/A represents it is not applicable to the permit action. Also included are the dates of any applicable Public Notice, Hearing Notice, the actual Hearing and the Comment Period End.

Some Permit Detail stages can be edited but others will be “locked” to prevent access to data changes. If a permit has not been issued as final and has not been dead-ended or denied, the Permit Detail page will be editable by clicking on the Edit button at the bottom of the page.

Please note, although permit-by-rule (PBR) is a possible search criteria from the Permit Search page, PBR details are not discussed in this Help page. Whenever selecting a PBR from a search, a Permit Detail page will not be generated. You will go directly to the Application Detail page whether the PBR is in process or has been reviewed. For information on details regarding a PBR, please refer to the Help page for Application Detail.

Also, historical Registration and State PTO Permit Detail pages will not be discussed since editing of these actions cannot occur. If you are viewing one of these actions and would like more information on the meaning of the fields you will see, refer to the “PTI and PTIO Permit Information”, “Terms and Conditions” and “Notes” headings below.

Duplicating Permits

The moment an application for a PTI or PTIO is submitted a workflow and Permit Detail page will be generated allowing DO/LAA staff to begin the permit process. Although applications may be submitted for multiple types of actions (initial install, modification, etc.) there are limitations in the type of actions that can be processed and issued in one PTI or PTIO permit document (this is not applicable to Title V permits). It may be necessary for Ohio EPA staff to split up the emissions units included in one application into multiple permits for issuance. As a general rule, each permitting action should have the same processing path and end result. The following actions will need to be separated into different permitting actions for processing:

  • Initial
  • Chapter 31 modifications (Ch. 31 mod)
  • Administrative modifications (admin mod)
  • Renewals
  • General permits

For example, general permits do not go through Central Office review. Therefore, it could not be combined with an installation permit that does require Central Office review. A separate Permit Detail page will need to be generated for each permit type emissions unit combination.

If it is determined that an application should be processed in more than one separate permit documents, then you will need to split the application into multiple Permit Detail pages resulting in multiple issued permits. This is accomplished on the Application Detail page by selecting the Duplicate Permit/Workflow button for each additional Permit Detail page needed. For assistance in completing this step please see the Application Detail Help page.

Once the multiple permit documents have been generated, open the Permit Detail pages for the duplicated permits. You will see that they also contain all of the emissions units, in associated (Title V permits) or disassociated (PTI and PTIO permits) form, that were included in the original application. You will need to include/exclude, or associate/disassociate, the emissions units, as described under the heading “Including and Excluding Emissions Units” below, so that each Permit Detail page contains the appropriate emissions units that will be issued in each of the final permit documents.

VERY IMPORTANT - when an application is submitted with multiple permit reasons (actions), the corresponding Permit Detail page will default the reason for the permit to the most stringent processing reason for all emissions units identified in the application. As you associate and disassociate emissions units in each of the Permit Detail pages you will need to review the application and correctly identify the corresponding reason for the emissions unit(s) included in each individual Permit Detail page. This is accomplished under the Reasons(s) check box at the top level of the Permit Detail page as discussed under the heading “PTI and PTIO Permit Information” below. It is very important the appropriate reason is correctly identified for each permit that will be issued from one application submittal.

Note: separating permit actions could result in different issuance and effective dates for emissions unit(s) which may be a part of the same project. DO/LAA staff will need to work with Central Office to identify if some actions should be issued together.

Facility Tree

You will notice the Facility Tree on the left hand side of all Permit Detail pages. Clicking any emissions unit ID in the Facility Tree will take you to an Emissions Unit page that allows you to edit, when applicable, or simply view emissions unit information. Recall, you can only edit information if the permit has not yet been issued. The information you see, and that will be editable, will also depend on whether the emissions units are associated (included) or disassociated (excluded) with the specific Permit Detail page.

  • Gray Gears with X -  – These emissions units are not part of, or they are “disassociated” with this permit.
  • Blue Gears -  – These emissions units are part of, or “associated” with, the permit being viewed.

When Ohio EPA receives, or enters, an application electronically, a corresponding Permit Detail page is generated. Only those emissions units that are included in the application (not always all emissions units at the facility) will be identified in the corresponding Permit Detail page as associated (Title V permits) or disassociated (PTI and PTIO permits) with the permit. You will review the application and determine if all requested emissions units should be associated with the permit and make the appropriate associations/disassociations. You may not associate emissions units with the permit that were not included in the original application submittal. This would required an amended/corrected application submittal. You may not need to associate all emissions units. This is necessary because there may be cases where not all emissions units will be issued in the same permit document as discussed above under the heading “Duplicating Permits.”

Including or Excluding Emissions Units

You may exclude (disassociate) an emissions unit for a permit not yet issued which will allow you to remove emissions units that will comprise the final permit. You may also include (associate) an emissions unit only if it was originally included in the application and then disassociated for some reason. Recall, when an application is submitted in the system it will include, and the corresponding Permit Detail generated from the application, all emissions units included in the application submittal. It may be necessary to include or exclude emissions units for various reasons. For example, several emissions units may have been part of a planned project but the facility later decided to not include one of the emissions units. Perhaps in the middle of the permit process the facility decided to permanently shutdown one of the emissions units included in a renewal application. Very importantly, it may be necessary for Ohio EPA staff to split up the emissions units included in one application into multiple permits for issuance (see the “Duplicating Permits” heading above for details). To include or exclude emissions units do one of the following:

  • Click on the Add/Remove EUs button at the bottom of the Permit Detail page. This button will take you to another page titled Permit Detail, Emissions Units. To add, select the emissions unit from the Available EUs and select Move to add it to the Current EUs. To remove, select the emissions unit from the Current EUs and select Remove to move it to the Available EUs. Don’t forget to select Save. To get back to the Permit Detail page click on the permit number hyperlink on the Facility Tree.
  • Select the emissions unit on the Facility Tree and click the Add EU to permit button or Exclude EU from permit button. To get back to the Permit Detail page click on the permit number hyperlink on the Facility Tree.

Emissions Unit Information

The following discusses the information you find on the Emissions Unit page and the fields that are editable when you are working with a permit that has not been issued. When an emissions unit is disassociated with the permit object,  , only the EU ID and EU Description will be visible. Emissions Unit pages for those units associated with the permit will contain the following information and will be editable, when applicable, via an Edit button available at the bottom of the page. Click Edit to make changes to the emissions unit information.

  • EU ID- The unique ID that is assigned to an emissions unit in the Facility Profile upon creation. This will be automatically included in the permit when the terms and conditions are generated and is not editable except in the Facility Profile. If it was necessary to update an EU ID in the Facility Profile AFTER this Permit Detail page was generated from the submittal of an application you will need to use the Sync EUs with Facility button to update the new EU ID number(s)(see below).
  • Company Equipment ID- The term the facility uses internally to identify the equipment. This is populated from the Facility Detail but can be edited, if necessary. This ID will be automatically included in the permit, when the terms and conditions are generated, as it is identified in this field.
  • EU Permit Status- Identifies the status of the permit, for example Expired or Rescinded. Extended indicates a timely renewal application was submitted but the renewal permit was not issued prior to expiration. If the emissions unit being created is new, the system will automatically default the EU Permit Status to None. The EU Permit Status will automatically be updated by the system as permits are issued or revoked. The only time this field should be edited is if a permit should be terminated, by selecting Terminate from the picklist. This would occur after final issuance and only the DO/LAA Supervisor will have the ability to edit in this case.
  • AQD Description- AQD description used to identify the equipment. This is populated from the Facility Detail but can be edited, if necessary. This ID will be automatically included in the permit, when the terms and conditions are generated, as it is identified in this field.This is not required in the system but is beneficial to differentiate between similar emissions units at the facility. When editing, this description should be short but specific.
  • Install Date- The date that construction or changes began. This field is not editable but is automatically updated by the system when the Begin Installation/Modification Date for the emissions unit is entered into the Facility Profile.
  • Terminated Date- A termination date can be entered when a PTI or initial installation PTIO has not been acted on for 18 months and no request for an extension has been made by the facility. This would occur after final issuance and only the DO/LAA Supervisor will have the ability to edit in this case.
  • General Permit Category - If a general permit is available and was applied for this emissions unit, the higher level category will be identified. For example, Readymix Concrete Batch Plants. If this emissions unit will be issued as a general permit but the category or type needs editing, you may select from a picklist when in edit mode. If it is determined to be not applicable, this may also be selected.
  • General Permit Type - If a general permit is available and was applied for, for this emissions unit, the higher level category will be identified. For example, Truck Mix (GP4.1). The designation in parenthesis is the unique identifier for that type of general permit. If this emissions unit will be issued as a general permit but the category or type needs editing, you may select from a picklist when in edit mode. If it is determined to be not applicable, this may also be selected.
  • Superseded Permit- Only available for PTI and PTIO permits (superseding for Title V permits occurs at the permit level as discussed below). If a previously issued permit was applicable to the emissions unit, the permit number is identified. The system will automatically default to None but if a previously issued permit was applicable to the emissions unit, the field will be editable and the permit number(s) will be identified in a picklist. Select the appropriate permit number that this permit will supersede. The superseded permit number will be identified in the issued permit so it is very important to identify this information.
  • Superseded Date- If this permit has been superseded by another permit for this emissions unit, the date will be identified after the superseding permit is issued final.
  • Fee section- Editable for installation/modification permits not issued final to identify the following fee information:
    • Category- Identifies the fee category that is applicable to the emissions unit.
    • Fee Name- The power or process weight rate levels associated with a Category type.
    • Adjustment- Designates any alterations to the Base amount, for example Half for an Administrative Modification.
    • Base Amount- Auto-calculated by the system, this is the designated fee charged to the owner or operator of a facility based on the Category and Fee name prior to any adjustments made to the fee.
    • Adjusted Amount- Auto-calculated by the system, the amount that the owner or operator of a facility will be billed upon final issuance of the PTI or PTIO (except when Full Cost Recovery is selected from the third-level menu Fee Summary).

    Note: these fees are not applicable to renewal PTIOs in addition to TV PTOs. The fields will be visible for a renewal PTIO Permit Detail page but if anything other than $0 is entered you will not be able to save the data.

To return to the Permit Detail page click on the permit number hyperlink at the top of the Facility Tree.

PTI and PTIO Permit Information

To edit information located in this section, click on the Edit button at the bottom of the screen. Many of the editable fields contain a check box. When editing, click on the text box to add or remove a check mark. Information contained in these fields and the ability to edit, when applicable, is discussed below.

  • Federally Enforceable PTIO (FEPTIO)- Only available for PTIO permits. When checked, this box indicates the emissions unit(s) in this permit will be part of a federally enforceable limitation on potential-to-emit necessitating issuance of an FEPTIO. If checked, an additional check box indicates if the purpose of the limitation is to avoid Title V applicability: FEPTIO to avoid Title V. These are editable fields.
  • Reason(s)- The Reason(s) for the permit is taken from the initial application submittal. If you need to change the reason; while in edit mode click on the checkmark next to the original designation to remove it and then reassign a reason from the other choices by clicking on the empty box. This will be necessary if you have duplicated the permit/workflow for issuance of multiple permits from one application (see the “Duplicating Permits” heading of this Help page).
  • General Permit- This option indicates if a general permit was applied for. This field can be edited if it is later determined during permit review that a general permit is not applicable, or likewise is available and applicable for the emissions unit(s) and the applicant desires a general permit. Keep in mind, the application submittal standards for general permits are not the same as a traditional permit. If the applicant applied for a general permit and will now be issued an individual permit, additional application information may be necessary. Similarly, if the applicant applied for an individual permit and it was discovered that a general permit is available, and desired by the applicant, a Qualifying Criteria document will also be necessary. If the applicability of a general permit is decided after the application was submitted and this box is checked, you will need to return to the emissions unit’s information and select a category and type. The category and type fields will only be available for editing at the emissions unit level AFTER this box is checked.
  • Express Permit- This option indicates the facility applied for express processing of their permit based on the criteria identified in the application. This field is editable if express processing is requested after application submittal or if it is determined to not be applicable.
  • Rush Permit- This option indicates that internally a decision was made to rush process the permit. This is not the same as express processing. This field is editable.
  • PER Due Date- Only available for PTIO permits. This is the annual reporting date that is permanently associated with the facility for all PTIO (including FEPTIO) permits issued to a facility. Normally, this will be the date requested by the facility during the first application process. This field is editable only for the first PTIO that will be issued for the facility, for example if the owner or operator requests a date and the DO/LAA determines another date is preferred. For subsequent PTIO's, this field will not be editable.
  • Description- This section is edited by the DO/LAA. The description should be chosen carefully because it will be transferred into the issued permit’s authorization page.
  • ERAC Case Number- This section is edited by the DO/LAA. If applicable, the case number as assigned by ERAC is entered if this permit is being issued to resolve an appeal.
  • Avoiding Major NSR Synthetic Minor- This section is edited by the DO/LAA if a synthetic minor restriction is applicable for an emissions unit that is avoiding major NSR only.
  • Modeling Submitted- This section is edited by the DO/LAA if modeling information was submitted.
  • Netting- This section is edited by the DO/LAA if netting was involved in the permit.
  • Toxics Review- This section is edited by the DO/LAA if a toxics review is being done as a part of this permit action.
  • Major Non-Attainment- Only available for PTI permits. This section is edited by the DO/LAA if the permit will be issued as a non-attainment major new source review permit requiring emissions offsets.
  • PSD- Only available for PTI permits. This section is edited by the DO/LAA if the permit will be issued as a prevention of significant deterioration (PSD) major new source review permit.
  • Variance- This section is edited by the DO/LAA if this permit will be issued as a variance.
  • CEM- This section is edited by the DO/LAA if one or more of the emissions units included in this permit will be monitored through a CEM.
  • MACT, NESHAPs, or NSPS (federal rules)- If one of the federal rules identified by the check boxes applies to the facility, and they are not already checked (from the application process), this section can be edited by the DO/LAA. If a check box is already selected and it is determined as part of the application review that the federal rule is not applicable, the check box can also be deselected. When adding certain federal rules it will be necessary to also identify applicable subpart(s) via a picklist generated when the box is checked. Select Add Subpart and select from the picklist. If a federal rule is shown as applicable and it is determined to not apply, select the subpart by placing a check mark in the box and click Delete Selected Subparts.
  • Hearing Requested- Indicates the public or Ohio EPA has requested that a public hearing be held. This is an editable field if it is determined to apply.
  • Permit Strategy Summary Write-up- Will contain a hyperlink to the synthetic minor (to avoid Major NSR) write-up, if applicable. If a write-up is not identified, you can add one by clicking on the Upload button. You will be required to enter a Description. This should be concise as it will become the hyperlink for opening the document once uploaded. You may upload a document multiple times; however, the previous document will always be replaced by the one being uploaded.
  • Response to Comments- Will contain a hyperlink to a response to comments document applicable to any comment period. If a document is not identified, you can add one by clicking on the Upload button. You will be required to enter a Description. This should be concise as it will become the hyperlink for opening the document once uploaded. You may upload a document multiple times; however, the previous document will always be replaced by the one being uploaded.
  • Application/Request Administrative Permit Modifications- This field is populated with application numbers for applications or Requests for Administrative Permit Modifications that are being addressed with the issuance of the permit. A datagrid will identify the application Number, Type and Purpose. You can associate other applications with this permit by entering a valid application number in the text box and clicking Add. The emissions units that are in the application(s) which is associated with the permit is what determines which emissions units are eligible to be included in the permit. In extremely rare circumstances an emissions unit can be added to a permit that is not included in the application (i.e., the data was unavailable to migrate for a very old permit); however, that must be done by an Administrator. For Help on generating terms and conditions, see the Help page Terms and Conditions. You can disassociate an identified application by checking the box next to the Number and clicking Delete selected.

    VERY IMPORTANT – If you receive a corrected/amended application (from the facility or internally) you MUST associate that application with THIS Permit Detail page. Facility corrected applications and AQD amended applications do NOT create a permit instance or a workflow. You will receive an notification on your ToDos list for any corrected/amended application and you can view the corresponding Application Detail page for the relevant information but you MUST ALSO add the application number to this field.

  • Public Notice Content- Identifies the text of the public notice for the permit and whether it is standard language or if it has been modified to customize the language. This field is editable. If you wish to customize the language in the text box select the Customized Wording radio button and make the necessary changes. You may also attach a separate public notice document by selecting the Customized Wording in Supplied File radio button. Add the document by clicking on the Upload button. You will be required to enter a Description. This should be concise as it will become the hyperlink for opening the document once uploaded. You may upload a document multiple times; however, the previous document will always be replaced by the one being uploaded. Customized verbiage undergoes review at Central Office.

Terms and Conditions

The section identifies Terms and Conditions associated with a permit that are generated by the permit writer. A datagrid will identify: Description, Last modified by (name of person), Last modified date and Issued as (e.g., draft, final). The terms and conditions can be viewed by clicking the Description that appears as a hyperlink. The file will be opened using the same software that was used to generate the terms and conditions. You can make changes to the terms and conditions but you must save the document to your hard/network drive and then upload the document, if the permit has not been issued final, through the third-level menu Terms and Conditions. Please see the Terms and Conditions Help page for additional information on uploading your document.

Bottom Page Buttons

  • Edit- allows you to Edit fields that are editable. While in edit mode only two buttons will be displayed at the bottom of the page, Save changes or Discard changes. Use these buttons before moving on to other screens in the system.
  • Sync EUs with Facility- This button is used to update the emissions unit IDs (e.g., B001) for the emissions units that will be issued in the permit to match those found in the current Facility Detail. If the permit writer gets an error during the DO/LAA sign off workflow task completion that indicated: "EU ID out of sync"; the permit writer should go to the Permit Detail page and click Sync EUs with Facility. Doing this will ensure the emissions unit IDs that will be issued in the permit match those found in the current Facility Detail and it will be necessary to do this if you receive the error message that your permit has emissions units ID's out of sync. This situation could occur when an emissions unit ID is updated/changed to a different ID than existed when the permit object was first generated.
  • Add/Remove EUs- Click on this button, select from a list and either Add EUs or Remove EUs that are associated with this permit. Please see the heading “Including or Excluding Emissions Units” on this Help page for more information on this feature.
  • Create EU Group- Allows you to group emissions units that will have identical terms and conditions for the purpose of writing one set of terms and conditions. Click Create EU Group and enter an EU group name for the set of emissions units. This name will appear in the permit document. To add emissions units to the group, select the emissions unit from the Available EUs and select Move to add it to the Current EUs. To remove, select the emissions unit from the Current EUs and select Remove to move it to the Available EUs. Select Save changes or Discard changes. You will automatically be returned to the Permit Detail page where you will see the grouped emissions units identified under a grouped icon on the Facility Tree.
  • Dead-end- This button is used to end the permit process, for example, if the facility is withdrawing an application. A dead-ended permit may be reactivated again but only by a systems administrator; therefore caution should be maintained. It will be necessary to cancel the workflow associated with a permit that is dead-ended. See the Help page for third-level menu “Cancel” of the “Workflow Diagram” second-level tab.
  • Deny Permit- This button is used to acknowledge a permit denial process will be initiated. Selecting this button will not automatically start a denial workflow, rather the permit writer will also need to select the denial process as part of the workflow task for the specific permit action. Once Deny Permit is selected, the Permit Detail page will then display an Un-Deny button (see below).
  • Un-Deny Permit- Only visible when the Deny Permit option was selected. Selecting this button allows you to reverse the denial process.
  • Change to PTI- Only available for PTIO permits. Select if you want to change the permit from a PTIO to a PTI. This would be applicable if the facility is in transition from the non-Title V program (PTIO) to the Title V program (PTI).
  • Change to PTIO- Only available for PTI permits. Select if you want to change the permit from a PTI to a PTIO. This would be applicable if the facility is in transition from the Title V program (PTI) to the non-Title V program (PTIO).

Third-level Menu

To the left of the Permit Detail page are the following third-level menu options and the activities they allow:

  • Workflow List- allows you to view the tasks and workflow associated with the specific permit object contained in the Permit Detail page. This page provides summary information in a datagrid including completion status, who is assigned to tasks, due dates, etc.
  • Terms and Conditions- allows you to generate a shell permit document to begin developing terms and conditions in. This is also where you will upload the completed permit document to be sent to Central Office for review and eventual issuance.
  • Carbon Copy- allows you to add contact information for any person who wishes to receive a copy of the permit at draft (if applicable) or final issuance.
  • Fee Summary- allows you to view a summary of all fees associated with the emissions units in the permit , view the total fee assessed for the permit and modify the fee total.
  • Attachments- allows you to attach documents to the Permit Detail page.
  • Notes- allows you to add new, edit existing, or delete notes associated with this permit.

The specific Help pages for each of the features above provides specific details on the information and functions contained in those pages.
